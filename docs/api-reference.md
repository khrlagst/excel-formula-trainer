## 1. Core Formula Evaluation Engine

These functions and classes form the core parser, tokenizer, and evaluation engine that parses and executes Excel-like formulas against a mock spreadsheet grid.

### `evaluateFormula(formula, grid)`
Parses and evaluates an Excel formula string against a provided grid state.

* **Parameters:**
  * `formula` (`string`): The formula string to evaluate (e.g., `"=SUM(A1:B3)"` or `"A1+B1"`). The leading `=` is optional.
  * `grid` (`Object`): A key-value map representing the spreadsheet grid, where keys are uppercase cell references (e.g., `"A1"`) and values are numbers or strings.
* **Returns:** (`number|string|boolean|Object`): The evaluated result of the formula.
* **Throws:** An `Error` if tokenization, parsing, or evaluation fails (e.g., `#NAME?`, `#VALUE!`, `#DIV/0!`, `#REF!`, `#N/A`).
* **Example:**
  ```javascript
  const grid = { A1: 10, B1: 20 };
  const result = evaluateFormula("=SUM(A1, B1) * 2", grid);
  console.log(result); // Output: 60
  ```

### `tokenize(src)`
Analyzes a raw formula string and breaks it down into an array of tokens.

* **Parameters:**
  * `src` (`string`): The raw formula string to tokenize.
* **Returns:** (`Array<Object>`): An array of token objects. Each token has the structure `{ t: string, v: any }` where `t` is the type (`"num"`, `"str"`, `"bool"`, `"cell"`, `"name"`, `"op"`, `"eof"`) and `v` is the parsed value.
* **Throws:** `Error` on unterminated strings or unexpected characters.
* **Example:**
  ```javascript
  const tokens = tokenize("SUM(A1, 10)");
  // Returns: 
  // [ 
  //   { t: "name", v: "SUM" }, { t: "op", v: "(" }, 
  //   { t: "cell", v: "A1" }, { t: "op", v: "," }, 
  //   { t: "num", v: 10 }, { t: "op", v: ")" }, 
  //   { t: "eof" } 
  // ]
  ```

### `class Parser`
A recursive descent parser that converts an array of tokens into an Abstract Syntax Tree (AST).

#### `constructor(tokens)`
* **Parameters:**
  * `tokens` (`Array<Object>`): An array of tokens generated by `tokenize()`.

#### `parseExpr()`
Parses an expression and returns the root node of the AST.
* **Returns:** (`Object`): The AST root node.

#### AST Node Types:
* **Number:** `{ type: "num", v: number }`
* **String:** `{ type: "str", v: string }`
* **Boolean:** `{ type: "bool", v: boolean }`
* **Cell Reference:** `{ type: "cell", v: string }`
* **Range Reference:** `{ type: "range", start: string, end: string }`
* **Unary Operator:** `{ type: "unary", op: string, operand: Object }`
* **Binary Operator:** `{ type: "bin", op: string, left: Object, right: Object }`
* **Comparison Operator:** `{ type: "cmp", op: string, left: Object, right: Object }`
* **Function Call:** `{ type: "func", name: string, args: Array<Object> }`

### `evalNode(node, grid)`
Recursively evaluates an AST node against the spreadsheet grid.

* **Parameters:**
  * `node` (`Object`): The AST node to evaluate.
  * `grid` (`Object`): The spreadsheet grid state.
* **Returns:** (`any`): The evaluated value of the node.
* **Throws:** `Error` if an evaluation error occurs.

### `formatValue(v)`
Formats a JavaScript value into its Excel-compatible string representation.

* **Parameters:**
  * `v` (`any`): The value to format.
* **Returns:** (`string`): The formatted string. Booleans are converted to `"TRUE"` or `"FALSE"`. Decimals are rounded to 6 decimal places.

### `matchesExpected(expected, actual)`
Compares an expected value against an actual value, allowing for floating-point precision tolerance.

* **Parameters:**
  * `expected` (`any`): The expected value.
  * `actual` (`any`): The actual value to test.
* **Returns:** (`boolean`): `true` if the values match, otherwise `false`.

## 2. Grid & Cell Utilities

These utility functions handle cell reference parsing, column coordinate conversions, and range extraction.

### `letterToCol(letters)`
Converts Excel-style column letters (e.g., `"A"`, `"Z"`, `"AA"`) to a 0-based column index.

* **Parameters:**
  * `letters` (`string`): The column letters (case-insensitive).
* **Returns:** (`number`): The 0-based column index (e.g., `"A"` -> `0`, `"B"` -> `1`).

### `colLetter(c)`
Converts a 0-based column index back to Excel-style column letters.

* **Parameters:**
  * `c` (`number`): The 0-based column index.
* **Returns:** (`string`): The uppercase column letters (e.g., `0` -> `"A"`, `27` -> `"AB"`).

### `parseCell(ref)`
Parses an A1-style cell reference into 0-based row and column coordinates.

* **Parameters:**
  * `ref` (`string`): The cell reference (e.g., `"C5"`).
* **Returns:** (`Object`): An object containing `{ col: number, row: number }`.
* **Throws:** `Error` (`#NAME?`) if the reference format is invalid.

### `getCell(grid, ref)`
Retrieves the value of a cell from the grid.

* **Parameters:**
  * `grid` (`Object`): The spreadsheet grid.
  * `ref` (`string`): The uppercase cell reference.
* **Returns:** (`any`): The cell value, or an empty string `""` if the cell is undefined.

### `rangeData(grid, startRef, endRef)`
Extracts a 2D array of cell values representing a rectangular range between two cell references.

* **Parameters:**
  * `grid` (`Object`): The spreadsheet grid.
  * `startRef` (`string`): The starting cell reference of the range (e.g., `"A1"`).
  * `endRef` (`string`): The ending cell reference of the range (e.g., `"B3"`).
* **Returns:** (`Array<Array<any>>`): A 2D array of values representing the rows and columns in the range.

### `gridBounds(grid)`
Calculates the maximum row and column indices present in a grid object.

* **Parameters:**
  * `grid` (`Object`): The spreadsheet grid.
* **Returns:** (`Object`): An object containing `{ cols: number, rows: number }` representing the 0-based maximum bounds.

### `buildGridTable(grid, bounds, targetCell)`
Generates an HTML `<table>` element representing the spreadsheet grid for rendering in the UI.

* **Parameters:**
  * `grid` (`Object`): The spreadsheet grid.
  * `bounds` (`Object`): The bounds of the grid as returned by `gridBounds()`.
  * `targetCell` (`string|null`): The cell reference that is the target of the current exercise (will be styled differently).
* **Returns:** (`HTMLTableElement`): The generated table element.

## 3. Evaluation Helpers

Internal helper functions used by the evaluation engine and built-in functions (`FUNCS`).

### `asNumber(v)`
Coerces a value to a number if possible.

* **Parameters:**
  * `v` (`any`): The value to coerce.
* **Returns:** (`number|null`): The coerced number, or `null` if coercion fails.

### `isTruthy(v)`
Evaluates whether a value is considered truthy under Excel rules.

* **Parameters:**
  * `v` (`any`): The value to evaluate.
* **Returns:** (`boolean`): `true` if truthy, `false` if falsy (e.g., `0`, `false`, `""`, or `"FALSE"`).

### `flattenNumbers(args)`
Flattens an array of arguments (which may contain raw values or range objects) into a flat 1D array of numbers, ignoring non-numeric values.

* **Parameters:**
  * `args` (`Array<any>`): The arguments to flatten.
* **Returns:** (`Array<number>`): A flat array of numbers.

### `flattenAll(args)`
Flattens an array of arguments (including range objects) into a flat 1D array of all values.

* **Parameters:**
  * `args` (`Array<any>`): The arguments to flatten.
* **Returns:** (`Array<any>`): A flat array of all values.

### `valuesEqual(a, b)`
Compares two values for equality under Excel rules (case-insensitive for strings, with floating-point tolerance for numbers).

* **Parameters:**
  * `a` (`any`): First value.
  * `b` (`any`): Second value.
* **Returns:** (`boolean`): `true` if equal, otherwise `false`.

### `valuesLessEq(a, b)`
Compares if value `a` is less than or equal to value `b`.

* **Parameters:**
  * `a` (`any`): First value.
  * `b` (`any`): Second value.
* **Returns:** (`boolean`): `true` if `a <= b`, otherwise `false`.

### `compareCriteria(cellVal, criteria)`
Evaluates a cell value against an Excel criteria string (e.g., `">5"`, `"<=10"`, `"<>Apple"`, or exact matches).

* **Parameters:**
  * `cellVal` (`any`): The value of the cell to test.
  * `criteria` (`string|number`): The criteria to evaluate against.
* **Returns:** (`boolean`): `true` if the cell value meets the criteria, otherwise `false`.

### `isEmpty(v)`
Checks if a value is empty (`""`, `null`, or `undefined`).

* **Parameters:**
  * `v` (`any`): The value to check.
* **Returns:** (`boolean`): `true` if empty, otherwise `false`.

## 4. State & Progress Management

These functions manage user progress, XP, streaks, badges, and sound settings using `localStorage`.

### `loadState()`
Loads the user's progress state from `localStorage`.

* **Returns:** (`Object`): The loaded state object, or a default state if none exists.

### `saveState()`
Saves the current global `state` object to `localStorage`.

### `touchStreak()`
Updates the user's daily streak based on the last active date.

### `awardBadge(id)`
Awards a badge to the user if they have not already unlocked it.

* **Parameters:**
  * `id` (`string`): The unique identifier of the badge (e.g., `"firstPractice"`, `"firstQuiz"`).
* **Returns:** (`boolean`): `true` if the badge was newly awarded, `false` if the user already had it.

### `playSound(type)`
Plays a synthesized sound effect using the Web Audio API.

* **Parameters:**
  * `type` (`string`): The type of sound to play (`"success"`, `"error"`, or `"win"`).

## 5. DOM & UI Helpers

Lightweight utilities for programmatic DOM manipulation.

### `el(tag, props, children)`
Creates and configures an HTML element.

* **Parameters:**
  * `tag` (`string`): The HTML tag name (e.g., `"div"`, `"button"`).
  * `props` (`Object`, optional): An object containing element properties, attributes, and event listeners:
    * `class` (`string`): Sets `className`.
    * `text` (`string`): Sets `textContent`.
    * `on` (`Object`): Event listeners map (e.g., `{ click: () => {} }`).
    * `dataset` (`Object`): Custom data attributes.
  * `children` (`HTMLElement|string|Array`, optional): Child elements or text nodes to append.
* **Returns:** (`HTMLElement`): The constructed HTML element.
* **Example:**
  ```javascript
  const button = el("button", { 
    class: "btn btn--primary", 
    on: { click: () => console.log("Clicked!") } 
  }, "Submit");
  ```

### `clear(node)`
Removes all child nodes from a DOM element.

* **Parameters:**
  * `node` (`HTMLElement`): The element to clear.

## 6. I18N & Localization Utilities

### `t(key, vars)`
Retrieves a localized string translation for the current language.

* **Parameters:**
  * `key` (`string`): The translation key.
  * `vars` (`Object`, optional): Template variables to interpolate into the translation string (e.g., `{ name: "John" }`).
* **Returns:** (`string`): The translated and interpolated string.

### `L(obj)`
Returns a localized version of a content object (e.g., a formula description or quiz question) by merging the current language's `i18n` overrides into the base object.

* **Parameters:**
  * `obj` (`Object`): The content object containing an optional `i18n` map.
* **Returns:** (`Object`): The localized content object.

# Excel Formula Trainer - Data API Reference

This document describes the data schema and structure exported by `data.js` in the Excel Formula Trainer project. This file serves as the primary content database for the application, containing levels, formulas, practice exercises, quizzes, and internationalization (i18n) configurations.

## Exported Constants

### `LEVELS`
An array of `Level` objects representing the training curriculum.

- **Type**: `Level[]`

---

## Data Types & Schemas

### `Level`
Represents a learning module or level in the trainer.

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `number` | Unique identifier for the level. |
| `name` | `string` | Display name of the level (e.g., "Basics", "Logic"). |
| `icon` | `string` | Emoji icon representing the level. |
| `tagline` | `string` | Short description of the level's focus. |
| `formulas` | `Formula[]` | List of Excel formulas taught in this level. |
| `practices` | `Practice[]` | Hands-on practice exercises for this level. |

---

### `Formula`
Represents an Excel function or operator taught within a level.

| Field | Type | Description |
| :--- | :--- | :--- |
| `name` | `string` | The standard name of the formula (e.g., "SUM", "VLOOKUP"). |
| `syntax` | `string` | The syntax signature of the formula. |
| `description` | `string` | English description of what the formula does. |
| `example` | `string` | An example usage of the formula. |
| `result` | `string` | English explanation of the expected result of the example. |
| `demo` | `Demo` | *Optional*. Interactive demo configuration. |
| `i18n` | `Record<string, FormulaI18n>` | *Optional*. Localized strings for supported languages (e.g., `es`, `id`, `fr`). |

#### `Demo`
Configures the interactive spreadsheet grid for demonstrating a formula.

| Field | Type | Description |
| :--- | :--- | :--- |
| `grid` | `Grid` | Key-value map representing the initial spreadsheet state. |
| `targetCell` | `string` | The cell coordinate where the formula should be evaluated (e.g., "A5"). |
| `formula` | `string` | The default formula pre-filled or evaluated in the demo. |

---

### `Practice`
Represents a hands-on practice exercise where the user must write a formula to achieve a target value.

| Field | Type | Description |
| :--- | :--- | :--- |

| `title` | `string` | Title of the exercise. |

| `grid` | `Grid` | Initial spreadsheet grid state. |
| `targetCell` | `string` | The cell where the user must enter their formula. |

| `solution` | `string` | The correct formula solution. |
| `i18n` | `Record<string, PracticeI18n>` | *Optional*. Localized strings for the exercise. |

---

### `Quiz`
Represents an assessment question. Quizzes can be multiple-choice (`mc`) or interactive written formula tasks (`write`).

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` | Unique identifier for the quiz question (e.g., "l1q1"). |
| `type` | `"mc" \| "write"` | The type of quiz question. |
| `prompt` | `string` | The question prompt or instructions. |
| `options` | `string[]` | *Conditional*. Array of choices. Required if `type` is `"mc"`. |
| `answer` | `number` | *Conditional*. The 0-based index of the correct option in `options`. Required if `type` is `"mc"`. |
| `title` | `string` | *Conditional*. Title of the task. Used if `type` is `"write"`. |
| `grid` | `Grid` | *Conditional*. Initial spreadsheet grid state. Used if `type` is `"write"`. |
| `targetCell` | `string` | *Conditional*. Target cell for formula entry. Used if `type` is `"write"`. |

| `solution` | `string` | *Conditional*. Correct formula solution. Used if `type` is `"write"`. |
| `i18n` | `Record<string, QuizI18n>` | *Optional*. Localized strings for the quiz. |

## Usage Examples

### Importing the Data
```javascript
import { LEVELS } from './data.js';

// Get total number of levels
console.log(`Loaded ${LEVELS.length} levels.`);
```

### Querying a Specific Formula
```javascript
const basicsLevel = LEVELS.find(level => level.id === 1);
const sumFormula = basicsLevel.formulas.find(f => f.name === 'SUM');

console.log(`Formula: ${sumFormula.name}`);
console.log(`Syntax: ${sumFormula.syntax}`);
console.log(`Description: ${sumFormula.description}`);
```

### Retrieving Localized Content
```javascript
function getLocalizedPractice(practice, lang = 'en') {
  if (lang === 'en' || !practice.i18n || !practice.i18n[lang]) {
    return {
      title: practice.title,
      prompt: practice.prompt,
      hint: practice.hint
    };
  }
  
  const localized = practice.i18n[lang];
  return {
    title: localized.title || practice.title,
    prompt: localized.prompt || practice.prompt,
    hint: localized.hint || practice.hint
  };
}
```

These constants define the core gamification rules and system settings.

* **Type**: `number`
* **Value**: `0.8`

* **Type**: `number`
* **Value**: `50`
* **Description**: The amount of Experience Points (XP) awarded to a user upon successful completion of a quiz.

### `XP_PER_PRACTICE`
* **Type**: `number`
* **Value**: `10`
* **Description**: The amount of Experience Points (XP) awarded to a user for completing a practice scenario.

### `LANG_ORDER`
* **Type**: `string[]`
* **Value**: `["en", "es", "id", "fr"]`
* **Description**: Defines the supported languages and their display/fallback order within the application.

### `LANG_NAMES`
* **Type**: `Record<string, string>`
* **Value**: 
  ```json
  {
    "en": "English",
    "es": "Español",
    "id": "Bahasa Indonesia",
    "fr": "Français"
  }
  ```
* **Description**: Maps language codes to their respective localized display names.

## Data Schemas & Models

### 1. `CONCEPTS` (Walkthrough Catalog)

The `CONCEPTS` array contains self-contained explanatory topics for the "Learn More" tab. Each concept object represents a specific topic (e.g., shortcuts, tables, charts) and conforms to one of several specialized schemas based on its `type` property.

#### Base Concept Schema
All objects in the `CONCEPTS` array share these base properties:

| Property | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` | Unique identifier for the concept topic. |
| `icon` | `string` | Emoji character representing the topic visually. |
| `type` | `"shortcuts" \| "walkthrough" \| "pivot" \| "chart" \| "powerquery"` | Discriminator field determining the layout and demo schema. |
| `title` | `string` | English title of the topic. |
| `intro` | `string` | English introductory description. |
| `i18n` | `Record<string, Partial<Concept>>` | Localization overrides indexed by language code (`es`, `id`, `fr`). |

---

#### Type-Specific Schemas

##### Type: `"shortcuts"`
Used to render categorized keyboard shortcuts.

* **Additional Properties**:
  * `groups` (`ShortcutGroup[]`): An array of categorized shortcut lists.

* **`ShortcutGroup` Schema**:
  * `name` (`string`): The category name (e.g., "Navigation").
  * `rows` (`ShortcutRow[]`): List of shortcuts in this group.

* **`ShortcutRow` Schema**:
  * `keys` (`string`): The key combination (e.g., `"Ctrl + Arrow"`).
  * `desc` (`string`): Description of what the shortcut does.

##### Type: `"walkthrough"`
Used for step-by-step text explanations accompanied by an interactive formula demo.

* **Additional Properties**:
  * `steps` (`Step[]`): Array of sequential steps.
  * `demo` (`TableDemo`): Interactive table configuration.

* **`Step` Schema**:
  * `title` (`string`): Step header.
  * `body` (`string`): Step explanation.

* **`TableDemo` Schema**:
  * `grid` (`Record<string, string \| number>`): Key-value pairs mapping cell coordinates (e.g., `"A1"`) to values.
  * `targetCell` (`string`): The cell coordinate where the formula is evaluated.
  * `formula` (`string`): The default formula populated in the target cell.
  * `note` (`string`): Explanatory note about the demo.

##### Type: `"pivot"`
Used for step-by-step explanations accompanied by a PivotTable simulation.

* **Additional Properties**:
  * `steps` (`Step[]`): Array of sequential steps.
  * `pivotDemo` (`PivotDemo`): PivotTable simulation configuration.

* **`PivotDemo` Schema**:
  * `source` (`Record<string, string \| number>`): Source data grid mapping.
  * `rowsField` (`string`): Field name assigned to rows.
  * `colsField` (`string`): Field name assigned to columns.
  * `valuesField` (`string`): Field name assigned to values.
  * `agg` (`"SUM" \| "AVERAGE" \| "COUNT"`): Aggregation function.
  * `note` (`string`): Explanatory note.

##### Type: `"chart"`
Used for step-by-step explanations accompanied by a chart visualization demo.

* **Additional Properties**:
  * `steps` (`Step[]`): Array of sequential steps.
  * `chartDemo` (`ChartDemo`): Chart configuration.

* **`ChartDemo` Schema**:
  * `kind` (`"column" \| "line" \| "pie" \| "bar"`): Default chart type.
  * `cats` (`string[]`): Category labels (X-axis).
  * `series` (`number[]`): Data points corresponding to categories.
  * `note` (`string`): Explanatory note.

##### Type: `"powerquery"`
Used for step-by-step explanations showing a "Before" and "After" data transformation state.

* **Additional Properties**:
  * `steps` (`Step[]`): Array of sequential steps.
  * `pqDemo` (`PQDemo`): Power Query transformation demo.

* **`PQDemo` Schema**:
  * `before` (`Record<string, string \| number>`): Grid mapping representing raw source data.
  * `after` (`Record<string, string \| number>`): Grid mapping representing transformed target data.
  * `note` (`string`): Explanatory note.

### 2. `I18N` (Localization Dictionary)

The `I18N` object contains translation dictionaries for all UI chrome, labels, tooltips, and feedback messages. It is indexed by language code.

* **Type**: `Record<string, Record<string, string>>`

#### Key Translation Tokens
Below are key tokens used by the rendering engine to inject dynamic values:

* `"level.chip"`: `"Level {id} · {name}"` (Interpolates current level ID and name)
* `"xp"`: `"XP {n}"` (Interpolates current XP points)
* `"learn.head"`: `"{icon} Level {id}: {name}"` (Interpolates level icon, ID, and name)
* `"invalid"`: `"⚠️ {msg}"` (Interpolates formula validation error messages)
* `"notQuite"`: `"❌ Not quite. Your formula returned {got}, but the expected result is {exp}."` (Interpolates evaluation results)

## Integration & Usage Example

To consume these data structures in an application module (e.g., `app.js`):

```javascript
// Retrieve localized level details
function getLevelDetails(levelId, langCode = 'en') {
  const meta = LEVEL_META[levelId];
  if (!meta) return null;
  return meta[langCode] || meta['en'];
}

// Retrieve localized UI text with interpolation
function getTranslation(key, langCode = 'en', variables = {}) {
  let translation = I18N[langCode]?.[key] || I18N['en']?.[key] || key;
  
  Object.entries(variables).forEach(([name, val]) => {
    translation = translation.replace(`{${name}}`, val);
  });
  
  return translation;
}

// Example usage:
```
